1. Occlusion Culling in Unity3D

Die Idee dahinten ist für Optimierungsgründen. Occlusion culling ist ein sehr bekannte und benutzte Technik im Spielentwicklung, um effizient viele Objekte erzeugen zu können ohne die gesamte Performance zu beschädigen.

Man rendert nur solche Objekte, die gerade gebraucht werden, d.h, nur solche, die vor die Kamera stehen. Damit spart man Speicher, sodass unser Spiel besser laufen kann.

Probieren Sie es selber, hier gibt ein kleiner Demo: A S W D sind die Kontroller. Oder besuchen Sie meine github: Unity3D Projekt

2. Rubiks Cube für die PSVita

Hier ist ein klein Spiel für die PSVita. Um es ausführen zu können, braucht man die Sony API im Rechner installiert zu haben und ein PSVita, Leider ist nicht einfach, die Lizenz dafür zu bekommen, trotzdem hier kann man sich die Projekt im github anschauen: Rubiks Cube

3. Web-Anwendungen für Bezirk Trier und Umgebung

Eine kleine modulare Web-Anwendung mit RESTful API für die Backend. Mit Hilfe von Leaflet (Open Source Karteanzeige) kann man durch die Region navigieren und sich die Höhenprofil anschauen.

Diese Web-Anwendung beinhaltet verschieden wichtige APIs so wie Best-Practices. Es wurde Modular mit Hilfe von Common.js konstruiert, die Backend benutzt RESTful API für die Strukturierung von Serveranfragen und Leaflet als Open-Source Karteanzeige. Github: Trier Karte

4. Google Chrome Erweiterung: Zeitmessung für die Browser

Diese kleine Erweiterung zeigt uns, wie viel Zeit man in die verschiedene Webseite investiert. Es war ein lustige Experiment. Obwohl es ein paar Bugs hat, funktioniert einwandfrei.

Diese Erweiterung wurde mit Hilfe von die Google Developer API und Chart.js gemacht. Github: Zeitmessung Erweiterung

5. Angular.js: Lebenslauf als Web-Anwendung

Zum Schluss ist diese Webseit auch als Open-Source zu Verfügung.

Dahinten sind viele Technologien, die heutezutage relevant sind. Node Package Manager und Node sind die Gründsteine des Servers, sie verwalten alle die Abhängigkeiten (Dependencies) und Bower verwaltet die Abhängigkeiten der Client. Für die Client habe ich Angular.js für DOM-Manipulation benutzt und D3.js für die graphische Darstellung von Daten. Im Backend wurde Express für die Server benutzt, sowie Socket.io für die Abfragen und Abschicken von Daten, die die Client benutzt. Github: AngularApp
